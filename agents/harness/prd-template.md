# PRD: Short Product Name or Feature

Status: draft

Allowed statuses: draft, approved, needs-review

This template becomes a concrete PRD when copied into a dated or named file and completed for a specific product, website, or feature.

## Product Goal

State the business or user outcome this work must achieve.

## Audience

Describe the primary audience, secondary audience, and any excluded users.

## User Jobs

- State the primary job the user needs to complete.
- State supporting jobs only when they affect scope or acceptance.

## Requirements

- List approved functional, content, UX, data, and operational requirements.

## Non-Goals

- List work that is intentionally outside this PRD.

## Acceptance Criteria

- State the observable conditions required for the work to be accepted.
- Include verification expectations for the intended task type.

## Open Decisions

- Record decisions that must be resolved before implementation can start.

## Source Links

- Link source files, wiki pages, briefs, or user decisions that support this PRD.
